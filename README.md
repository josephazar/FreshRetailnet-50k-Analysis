# FreshRetailNet-50K: Demand Forecasting & Inventory Optimization

End-to-end pipeline that forecasts demand for perishable retail products and optimizes order quantities using Newsvendor theory. Built on the FreshRetailNet-50K dataset (50,000 store-product combinations, 90 days of sales).

## Key Concepts

- **Censored demand recovery** — sales ≠ demand when stockouts occur (~44% of observations); Tobit/IMR correction used as input feature
- **120+ engineered features** — rolling stats, EWMA, day-of-week profiles, hierarchy clustering, lag features
- **LightGBM ensemble** — MAE + Huber + quantile models with 4-fold expanding-window temporal CV
- **Newsvendor optimization** — Normal, Gamma, Empirical, and KDE distribution policies evaluated on a profit–service Pareto frontier
- **ABC-XYZ segmentation** — prioritizes modeling effort on high-value, predictable items

## Quick Start

```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# Run the full pipeline (~10 min on 3K SP sample)
python demand_forecast_and_inventory_optimization.py

# Or explore step-by-step via notebooks
jupyter lab notebooks/
```

## Data

Place the FreshRetailNet parquet files at `../../data/freshretailnet/raw/data/{train,eval}.parquet` relative to the notebooks directory, or adjust the paths in the pipeline script.

## Requirements

Python 3.10+ — NumPy, Pandas, LightGBM, scikit-learn, SciPy, Matplotlib, Seaborn, Plotly.
